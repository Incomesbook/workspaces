# AI_MEMORY

## Permanent memory for future AI agents

Igor wants one reliable J-based work environment where projects, AI chats, context, tools, and restore instructions are preserved.

The main goal is continuity:
- continue work from the last state
- preserve years of AI conversations
- preserve project decisions
- allow different LLMs/AI agents to understand the same project
- restore on another computer if needed

## Important rules

- Always read MASTER_STRUCTURE_AND_MEMORY_LOCK.md first.
- Never delete files blindly.
- Never move projects blindly.
- Never push secrets.
- Never push raw huge chats to normal Git.
- Never break existing Codex / Claude / VS Code automation.
- Use manifests, indexes, restore plans, and AI Context Capsules.

## Known issue

Native app chat history is not the same as GitHub backup.
Web/Desktop chats require export/import planning.
